# AGENTS.md

本仓库用于建设一套面向学生的 Python annotated math 课程：通过编写和阅读 Python 代码，帮助学生理解数学教材中的概念、解题思路与可视化表达。

## 课程定位

- 目标对象：当前从六年级数学开始，参考资料放在 `reference/`。
- 核心方法：把数学问题拆成可运行、可观察、可修改的代码流程，让学生理解“为什么这样想”和“每一步在做什么”。
- 内容目标：重视思维理解和迁移能力，难度从基础逐步提升到奥数水平，但不出偏题、怪题。
- 场景设计：题目应贴近真实生活，并尽量服务于后续计算机、数据科学与人工智能学习。

## 写作风格

- 采用 annotated notebook 风格：代码、图示、推导和讲解交替出现。
- 讲解要通俗、准确、循序渐进，参考李永乐老师式的直观表达。
- 可视化要清晰、美观、有解释力，参考 3Blue1Brown 式的概念呈现。
- 避免只给答案。每道题都应说明观察、建模、推理、验证和反思过程。
- 中文表达要适合学生阅读：短句优先，少用术语；必须使用术语时，先给直观解释。

## Notebook 设计

- 每个 notebook 对应一个小节，预计学习时间不超过 15 分钟。
- 每节只聚焦一个核心概念或一种解题方法，避免堆叠太多目标。
- 推荐结构：
  1. 问题情境：从教材题、生活问题或可视化现象引入。
  2. 思维视角：说明本题主要训练哪类思维，例如演绎、归纳、类比、分类讨论、化归、逆向思维、构造或数形结合。
  3. 直观理解：用图形、表格、数轴、面积、动画或枚举帮助观察。
  4. Python 建模：把数学对象转成变量、列表、函数或图形。
  5. 解题过程：逐步运行代码，并解释每一步对应的数学含义。
  6. 变式练习：改变数据或条件，让学生看到规律是否仍成立。
  7. 小结反思：总结方法、适用条件和常见误区。

## 代码要求

- 示例代码优先使用标准库、`numpy`、`matplotlib` 等学生容易理解的工具。
- 代码应短小、可运行、可修改。变量名要表达数学含义，避免炫技。
- 复杂代码需要拆成小函数，并在 notebook 中解释输入、输出和关键步骤。
- 可视化应服务于理解，不做无关装饰；图像标题、坐标轴和标注要清楚。
- 保持 notebook 输出可复现，避免依赖随机结果；如需随机数，设置固定 seed。

## 难度与题目原则

- 从教材基础概念出发，逐步加入更开放的思考题。
- 奥数级内容应强调方法来源和结构理解，例如分类讨论、枚举、逆向思维、数形结合、极端情况和不变量。
- 讲解解题思路时，可以从思维方式角度给题目分类，帮助学生识别“这类题为什么这样想”。
- 常用分类包括：演绎推理、归纳发现、类比迁移、分类讨论、逆向思维、数形结合、枚举验证、构造与不变量。
- 思维分类应服务于理解，不要变成抽象标签；每次标注后都要用学生能懂的话说明它在本题中的作用。
- 不追求刁钻技巧。题目应让学生通过观察、实验和推理自然接近解法。
- 每个难题都应提供可视化、代码实验或分步提示，避免直接跳到结论。

## 协作原则

- 修改内容前先查看现有目录、Notebook 风格和参考资料，尽量延续已有结构。
- 新增课程时优先创建小而完整的 notebook，不要一次写成过长章节。
- 涉及教材内容时，应对照 `reference/` 中的资料，保持年级、章节和术语一致。
- 生成或修改 notebook 后，尽量运行构建命令检查 Jupyter Book 是否能正常生成。
